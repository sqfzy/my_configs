# Issue report template

## 使用说明

Issue 报告用于**非事故性质但需要升级的问题**：卡点、需要跨团队协作、预期外的偏差、风险信号。与 incident 的区别：issue 不一定有 SLA 破线、不一定有用户感知，但需要外部关注/资源。

**适用场景**：需要资源协调的阻塞、发现但未爆发的问题、合规/安全隐患、跨团队依赖卡点
**不适用**：已造成生产影响（用 incident）、routine 风险（放进 progress 报告）

---
