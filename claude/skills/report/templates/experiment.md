# Experiment report template

## 使用说明

实验报告记录**一次以证伪为目的的验证**。核心价值不是"我做了什么"，而是"我们对世界多了什么确定性"——即使结论是"数据不足"，也是有价值的产出。

读者可能是决策者（据此做产品/技术决策）、研究者（复用方法论）、未来的你（避免重复实验）。因此报告必须让人能**独立判断证据强度**，而不是被结论牵着走。

**适用场景**：A/B 测试、性能假设验证、算法评估、产品实验、技术可行性 spike、POC
**不适用**：常规 benchmark（用 bench 产出）、成果汇报（实验已成功 → 用 achievement）
