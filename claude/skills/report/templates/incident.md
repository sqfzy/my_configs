# Incident report template

## 使用说明

事故报告 (Postmortem) 有**三重读者**：(1) 当事团队——复盘学习；(2) 其他团队——防止同类事故；(3) 高层/外部——问责与合规。三类人关注点不同，报告必须让任何一类都能独立读懂。

事故报告是**无责文化（blameless）**的载体——关注系统缺陷与流程漏洞，不追究个人。

**适用场景**：生产事故、SLA 违约、数据泄露、安全事件、重大运维故障
**不适用**：非生产问题（用 issue）、routine 运维事件（走工单系统）

---
