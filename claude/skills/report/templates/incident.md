# Incident report template

```markdown
# <事故标题> 事故报告

## 执行摘要
<1-3 句：发生了什么 + 当前状态（已恢复 / 仍在处理）+ 影响范围>

## 时间线
| 时间 (UTC) | 事件 |
|-----------|------|
| HH:MM | 首次异常发生 |
| HH:MM | 告警触发 / 人工发现 |
| HH:MM | 确认影响范围 |
| HH:MM | 实施缓解措施 |
| HH:MM | 完全恢复 |

## 影响范围
<受影响用户数 / 请求数 / 持续时长 / 错误率 / 业务损失>

## 根因
<为什么发生——技术根因 + 流程/制度层面的深层原因>

## 已采取措施
| 类别 | 措施 | 状态 |
|------|------|------|
| 立即止血 | ... | 已完成 |
| 短期修复 | ... | 进行中 |

## 长期改进
<防止同类事故再发生的结构性改进：监控/测试/流程/架构>

## Q&A
```
