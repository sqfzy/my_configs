# Progress report template

## 使用说明

进度报告的读者是**无法实时感知项目状态的人**（上司、跨部门协作方、远期利益相关人）。他们每周/每月只看这一份，据此判断要不要介入、要不要调资源。因此报告必须**独立完整**——任何一节被单独截图转发，都要能看懂。

**适用场景**：周报 / 月报 / 里程碑进度 / Sprint 总结
**不适用**：事故汇报（用 incident）、决策请求（用 decision）

---
