## 核心决策

新建 skill `/autopilot`（或 `/unattended`）——契约驱动 + checkpoint 持久化 + 分级 wakeup + 显式授权的长任务无人值守框架。

## 方案细节

### Phase 0：环境收集
- 读构建配置、git 状态、已有 `.artifacts/`
- **重入保护**：检测是否有同名 task 的 state 文件，若有则询问恢复/重启/查看报告

### Phase 1：快速验证 + 契约对话（≤5 分钟硬约束）

- **前 2 分钟**：环境探测 + 成功判据可判定性检查（并行）
- **后 3 分钟**：预填模式契约对话
  - Claude 根据任务类型预填 80% `INTENT.md`
  - 用户只审核 + 补结果敏感参数 + 补红线特例 + 授权条款细化
- **产物分裂存储**：
  - `.artifacts/autopilot-intent-<task>-summary.md`（≤100 行，wakeup 必读）
  - `.artifacts/autopilot-intent-<task>-full.md`（完整条款，按需 grep）

**INTENT.md 骨架**：

```markdown
# INTENT.md

## 1. 任务本体
- 一句话目标：
- 成功判据（可 shell 判定）：
  - [ ] 判据 A（命令 + 期望输出）
- 预计耗时：
- 启动命令：

## 2. 事实前提（预检已确认）
- [ ] 依赖 X 已安装，版本 Y
- [ ] 磁盘空间 > Z
- [ ] Claude Code 主进程需保持运行（wakeup 前提）

## 3. 授权清单（L2 操作必须显式列出）
- [ ] 重启挂掉的子进程
      条件：exit code ∈ {非 0, 非 SIGTERM}，且 restart_count < 3
      动作：以原命令重启
      上限：3 次

## 3.5 结果敏感参数（skill 不得自主调整）
- batch_size（调整会破坏实验可重现性）
- learning_rate, seed, ...

## 4. 红线（必须立即停下报人）
- git push / force push
- rm 任何 .artifacts 外文件
- 修改 INTENT.md 自身
- ...

## 5. 软停下（升级报人但不强制终止任务）
- 同一类错误连续 N 次
- 进度 X 分钟无推进
- 外部 API 报错率 > 阈值

## 6. 汇报协议
- 最终报告：.artifacts/autopilot-report-<task>.md
- 异常事件在报告顶部 `## ⚠️ 需要用户关注` 标记
```

**硬约束**：
- **A**：成功判据必须可 shell 判定（提供 5 种模板：exit-code / file-exists / grep-match / line-count / numeric-threshold）
- **B**：预检明确告知用户运行时要求（Claude Code 主进程保持运行）

### Phase 2：启动 + 初始密集检查

- `Bash(run_in_background=true)` 拉起任务
- PID + stdout 路径 + intent-version 写 `state.json v0`
- `ScheduleWakeup(60s)` 进入**启动保护期**
- 主会话当前轮结束

### Phase 3：Wakeup 监控循环

每次 wakeup 读取（≈200 行）：
- `intent-summary.md`（≤100 行）
- `state.json`（最新）
- 分类器最新输出（≤30 行）
- 日志尾 50 行

决策树：

```
1. 运行确定性红线检查（grep-based，不依赖 Claude 判断）
   红线命中 → 终止任务 → 写报告 → 退出循环

2. 运行分类器提取事件
   事件 = 正常 → 按节奏模板决定下次 wakeup
   事件 = warning → 切密集检查
   事件 = 错误
     ├─ 匹配授权条款 → 执行动作（按 "保护进度 > 最小破坏 > 最保守" 选策略）
     │                → 写 action record 到 state.json
     │                → 下次 wakeup 密集检查
     ├─ 匹配软停下条件 → 标 escalate=true → 报告中标记 → 继续观察
     └─ 无匹配 → 停下升级报人
```

**Wakeup 节奏模板**（硬编码，非动态决策，避免漂移）：

| 阶段 | 间隔 | 触发 |
|------|------|------|
| 启动保护期 | 60s | 任务启动后前 5 分钟 |
| 稳定期 | 1200–1800s | 5 分钟后无异常 |
| Warning 密检 | 270s × 3 次 | 检测到 warning |
| 收尾期 | 600s | 进度 >90% |

**禁用区间**：绝不选 300s（缓存 TTL 陷阱——付了 miss 成本又没摊薄）。

### Phase 4：终止 + 汇报

- 触发：成功判据命中 / 任务退出 / 红线终止 / 用户介入
- 生成 `.artifacts/autopilot-report-<task>.md`
- 顶部 `## ⚠️ 需要用户关注` 章节集中所有异常事件、自主处置、软停下记录
- 软停下期间报告持续刷新
- 建议用户根据本次 lessons 更新分类器和授权清单

### 配套资源

- `shared/autopilot-classifier.sh` — 通用事件分类器（OOM / 退出码 / panic / 磁盘 / 网络）
- `shared/success-verifier.sh` — 成功判据验证模板
- `shared/autopilot-intent-template.md` — INTENT.md 预填模板
- `shared/autopilot-aware.md` — 其他 skill 感知运行中的 autopilot（避免冲突）

## 已解决的分歧

| 分歧点 | 解决方式 | 关键论据 |
|--------|----------|----------|
| 自主度上限 | 两层 L1/L2 + 显式授权清单 + 结果敏感参数黑名单 | R8 撤回了"L2 默认执行" |
