## 最终方案

### 3 个可并行只读分析 Agent

| Agent | 职责 | 输入 | 输出 | 使用者 | ROI |
|-------|------|------|------|--------|-----|
| review-analyzer | 代码质量多维审查 | scope + diff/path + focus | 问题清单 + VERDICT | review, ship, improve | 高 |
| test-scanner | 测试覆盖缺口分析 | scope + diff/path | 覆盖缺口表 + VERDICT | test, ship, improve | 高 |
| bench-runner | 性能基线对比 | bench_cmd + baseline_ref | 对比表 + VERDICT | bench, ship, improve, refactor, migrate | 高 |

### 标准输出格式
```
## AGENT RESULT
### AGENT: <name>
### VERDICT: PASS | FAIL | SKIP | INFO
### SUMMARY: <一句话>
### METRICS: <key=value pairs>
### DETAILS
<agent 特定内容>
```

### 编排模式
Skill 自行编排（模式 A），在 SKILL.md 的分析阶段加"并行启动"指令。

### 典型并行组合
| Skill | 并行启动 | 时机 |
|-------|---------|------|
| /ship | review-analyzer + test-scanner + bench-runner | Gate 1-3 |
| /improve | review-analyzer + test-scanner + bench-runner | Phase 1 |
| /review | review-analyzer | 核心流程 |
| /test | test-scanner | Phase 0 |
| /refactor | review-analyzer + bench-runner | Phase 0+4 |

### 第二批候选
- arch-analyzer（架构分析）
- doc-scanner（文档审计）

### 未解决
- bench-runner 的编译锁并行约束需用户决策
