## 最终方案

### 核心决策
轻量扩展 `/evolve` 扫描维度（新增代码质量、性能、文档），推进时委派给对应 skill。配合 `/repeat` 持续自驱动。

### 改动
1. evolve Phase 2.1 新增 3 个扫描维度（代码质量、性能、文档）
2. evolve Phase 2.2 缺口清单新增"推荐 Skill"列
3. evolve Phase 3 推进时根据推荐 skill 委派执行

### 使用方式
```
/repeat /evolve auto implement [duration: 4h]
/repeat /evolve auto implement [until: 连续2轮无新缺口]
```
